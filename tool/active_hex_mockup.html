HexTrail — Active hex 3D-feel mockup

All effects use existing PolygonAnnotation API. ~1Hz update rate to match battery budget.

0. Current (baseline)

Today: flat fill, white outline, identical to captured hexes.

E. Drop shadow halo (static)

Single offset blurred polygon underneath. Zero animation cost. Adds depth.

B. Pulse ring (1Hz)

Outline polygon scales + fades outward. Two staggered rings for richness.

★ Recommended: B + C + E combined

Drop shadow + breathing fill + slow outward pulse ring. ~1% extra battery.

Implementation notes

  • Pulse ring: new _currentHaloMgr PolygonAnnotationManager. Timer at 1.2s. Each tick: delete + re-create halo polygon at 1.0× → 1.45× scale (interpolated in 4 steps to fake smoothness, or just two-state on/off like the existing recommended-tile pulse).
  • Breathing fill: free — same timer toggles _currentPoly's opacity 0.55 ↔ 0.85 by recreating with alternating opacity values.
  • Drop shadow: single static polygon redrawn only when _currentTile changes. Offset ~4px right, ~8px down. Black @ 55% opacity. Zero per-frame cost.
  • Lifecycle hardening: add WidgetsBindingObserver to pause timer on AppLifecycleState.paused. Apply same pause to existing _recommendedTilePulseTimer as a free win.
  • Battery budget: 1Hz delete+create on a single polygon = ~2 platform-channel calls/sec. Existing recommendation pulse already runs at this rate without complaints. Estimated incremental drain: ~1% per active hour.
